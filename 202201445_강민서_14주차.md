## 1. 서비스 개요

### 서비스명
**MeetSync** — 구글 캘린더 연동 미팅 예약 서비스

### 서비스 설명
MeetSync는 Cal.com을 참고한 미팅 예약 스케줄링 서비스이다.
호스트(주최자)가 구글 캘린더와 연동하여 가능한 시간대를 설정하면,
게스트(참여자)는 로그인 없이 전용 예약 페이지에 접속해 원하는 시간을 선택할 수 있다.
예약이 확정되면 호스트의 구글 캘린더에 자동으로 일정이 추가되며,
온라인 미팅의 경우 Google Meet 링크가 자동 생성된다.

### 주요 기능 요약
- 구글 SSO 로그인 및 캘린더 권한 연동
- 호스트 가용 시간 설정 및 미팅 타입 관리
- 구글 캘린더 기반 실시간 슬롯 관리
- 게스트 예약 페이지 (로그인 불필요)
- 예약 링크 및 QR 코드 공유
- 온라인(Google Meet) / 오프라인(카카오맵) 미팅 장소 설정
- 예약 재스케줄 및 그룹 예약 지원
- 예약 현황 대시보드

## 2. 기술 스택

### Frontend
| 항목 | 내용 |
|------|------|
| 프레임워크 | React 18 (Vite) |
| 라우팅 | React Router v6 |
| 스타일링 | Tailwind CSS |
| 배포 | Vercel |

### Backend
| 항목 | 내용 |
|------|------|
| 프레임워크 | NestJS (Node.js + TypeScript) |
| ORM | Prisma |
| 인증 | Passport.js (Google OAuth2 + JWT) |
| 배포 | Render (Docker 컨테이너) |

### 데이터베이스
| 항목 | 내용 |
|------|------|
| DBMS | PostgreSQL |
| 호스팅 | Render (무료 플랜) |

### 외부 API
| API | 활용 목적 |
|-----|----------|
| Google OAuth2 | 소셜 로그인 (SSO) |
| Google Calendar API | 기존 일정 조회, 예약 이벤트 자동 추가 |
| Google Meet | 온라인 미팅 링크 자동 생성 (Calendar API 내장) |
| 카카오맵 API | 오프라인 미팅 장소 검색 및 지도 표시 |

### 주요 라이브러리
| 라이브러리 | 용도 |
|-----------|------|
| googleapis | Google Calendar / OAuth2 연동 |
| @nestjs/passport | 인증 미들웨어 |
| jsonwebtoken | JWT 발급 및 검증 |
| prisma | PostgreSQL ORM |
| qrcode | QR 코드 생성 |
| dayjs | 날짜/시간 처리 |

## 3. Web Storage 및 DBMS 활용 전략

### Web Storage
| 종류 | 활용 |
|------|------|
| `localStorage` | JWT 토큰 저장 (로그인 상태 유지) |
| `sessionStorage` | 예약 진행 중 선택한 날짜/시간 임시 저장 (새로고침 복원) |

### DBMS (PostgreSQL) 테이블 구성
| 테이블 | 주요 컬럼 |
|--------|----------|
| `users` | id, email, name, picture, timezone, googleRefreshToken |
| `event_types` | id, userId, title, duration, bufferTime, maxGuests, slug |
| `availability` | id, userId, dayOfWeek, startTime, endTime |
| `bookings` | id, eventTypeId, guestName, guestEmail, startTime, endTime, status, googleEventId |

## 4. Reverse-Proxy 및 Docker 활용 계획

### 배포 아키텍처
```
[사용자]
  │
  ├─ React 프론트엔드 → Vercel (CDN 배포)
  │
  └─ API 요청
        ↓
   [Render]
   Nginx (Reverse Proxy, 포트 80)
        ↓
   NestJS 서버 (포트 3000)
        ↓
   PostgreSQL (Render DB)
```

### Docker 구성
- `backend/Dockerfile` : NestJS 앱 멀티스테이지 빌드
- `nginx/nginx.conf` : 리버스 프록시 설정
- `docker-compose.yml` : NestJS + Nginx 컨테이너 묶음

## 5. CI/CD 계획

### GitHub Actions 자동 배포
```
GitHub main 브랜치 push
    ↓
[Frontend] Vercel 자동 배포
[Backend]  Docker 이미지 빌드 → Render 자동 배포
```

### Workflow 구성
- `.github/workflows/deploy.yml`
- 프론트엔드: Vercel Action 사용
- 백엔드: Docker Hub 푸시 → Render Deploy Hook 호출
